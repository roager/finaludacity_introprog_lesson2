I. Learn to Code

1. Whats is HTML

HTML is a language made of:
  • text content
  • markup
  • references
  • other documents
  • links to another pages

2. HTML tags

HTML tags usually are structures which tag content.
Almost all content is inside an open tag and a close tag.
Even some tags with content could be inside other tags.
<TAG> CONTENT</TAG>

3. Bold tags

Bold tags follow generic rules.
You open a tag, then put the content and then close the tag.
the result is bold text for selected content.
e.g. My content is <B> Bold</B>
the result is: My content is Bold

4. Italics

Italics tags follow generic rules.
You open a tag, then put the content and then close the tag.
The result is italics text for selected content.
e.g. My content is <em> in Italics</em>
the result is: My content is in Italics

5. Missing end tag

Computers don't understand instruction if you don't follow the rules.
If you forget to close you tags it will fail
e.g. <em> My tag is not closed
The result is: My content is in Italics
As you can see instruction for italics continue for all text.

6. Making Links

We have tags to create hyperlinks on text or images.
Link rules follow generic rules.
You open a tag, then put the content and then close the tag.
However, this adds an attribute for the link.
e.g. <a href="www.latinfy.com"> Link to Latinfy </a>
The result is: Link to Latinfy

8. Whitespaces

Whitespaces are not interpreted by HTML.
So if you let doouble space on a line one of them will be ignored.
And if you don't use multiple lines tags, then text will continue in a single line.
For that it is good to use tags like BR and P.
e.g. My text have BR tags < BR> This creates order.
The result is:
My text have BR tags
This creates order.

P tag is very similar, but this one requires open and close tags. e.g. < P> My text has P tags < /P> This creates order.
The result is:

My text has P tags

This creates order.

9. Inline and Block elements

The difference between inline and block tags is that Block tags create a kind of square around content which separate elements. Inline content elements just continue as a unified block, while block elements are like a subelement inside the big block. Examples of inline content are the next tags: a, span, br, img, and strong. Examples of block elements are: div, p and form.

II. Make a Stylish Webpage

1. HTML and CSS

Why to start with HTML and CSS:

  • It is fun to work with HTML and CSS.
  • HTML and CSS are very visual. Anything you write will appear inside of browser.
  • It is very helpful in order to learn other languages.

2. Web Analogies

HTML language is like a house 🏠 :

  • HTML files are like the structure, where files are like rooms.
  • CSS files are then the style of the house (carpets, decoration, colors, etc.).
  • Javascript are like the interctive components of the house (remote controls, televisions, etc.).

This elements just like in real life respond to rules, which in this case is the code.

3. Developer Tools


Actually browsers have different tools that can help us to analyze code structures at the same time we are browsing a page.
Developers tools read HTML code in a tree structure in order to undrstand the hierarchy of tags and find faster a logic on data.
Some tags can store other tags or elements inside of them, that is why is helpful to use these tools.
Develope tools can also find customization on each element, this customization is usually stored on CSS files.

4. HTML and CSS

In synthesis these are the rules:

  • HTML and CSS are both "languages"
  • HTML controls the "structure" of a web page
  • CSS controls the "style" of a page (how it looks).
  • When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page.

Now in more technical words:

  • HTML - HyperText Markup Language - the standard markup language used to create web pages.
  • CSS - Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.
  • DOM - Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

5. Boxes, Grids and Boxes

Elements in HTML are defined as squares.
This very useful because:

  • It easy to put everything in a right place.
  • this help to apply styles to each element.
  • It allows to define structures of data and design from abstract thinking.
Other great use for boxes is to envolve text inside divs, in this way elements have this useful features:
  • Hierarchy is easy to understand.
  • Classes can be attached easily with specific styles for elements inside DIV tag.
  • It allows to define structures of data and design from abstract thinking.

6. Prepare for CSS

In order to use CSS you need to assure that your HTML has the most important tags:

  • Doctype,HTML, head, title and body tags at the top.
  • The most recommended way to use CSS is to reference a CSS file to our HTML. Usually a line like this with the location of CSS file. <link rel="stylesheet" href="style.css">
  • It is recommended to use DIVs in the element where we will change format. This is in order to create classes attached to them.


We just need to add class attribute to the DIV tags where we want to add a defined style.
From here you will be able to create your CSS file rules.

7. CSS Hierarchy

CSS or Cascading Style Sheets explain HTML how to diaplay the items it contains. For this there are main rules that are pre defined for most of elements. However, when we use CSS we can specify situations when rules can change. For that we can say that CSS is more specific.

8. CSS First Styles

There are a lot of styles in CSS, but as there are many the plan is just to know more populars, how to find and apply them. These styles can include text formatting rules, useful to change text to bold, italics, color, size just to mention some.

9. CSS Vocabulary

Most important names to learn:

  • CSS: Cascading Style Sheets.
  • Style: This word can refer to many things and so it can be confusing. It can refer to:


The HTML element. For example:


<style>div {color:blue} </style>


The HTML attribute. For example:


<div style="color:blue" > this text would be blue </di >


The general look of a web page. For example: "I like that site's style."
A verb. For example: "I'm making progress on my page. The structure is all done but now I have to style it."

  • Rule: a line of CSS code describing the value that a certain attribute should take.
  • Property: The property you want to change.
  • Value: The value that you want to assign to the attribute.
  • Selector: The name that you use to in order to target the elements that are assigned to a class or id attribute in the HTML.
  • Class: A class refers to a group of elements that can be styled together. Class names should not contain periods or any other punctuation marks such as class="1.1"
  • ID: ID's are unique identifiers that uniquely identifies an element in HTML.

10. CSS Methods

There are three ways to apply CSS styles:

  • Method 1: Write CSS in the <head> of your HTML

<head> <style> div { background-color : red; } </style> </head> <body> <div> This will have a red background. </div> </body>
  • Method 2: Link your HTML to a separate CSS file

you would have a main.html file that looked like this:

<head> <link rel="stylesheet" href="main.css"> </head> <body> <div> This will have a red background. </div> </body>

and in the same folder you would have a file called main.css like this:

div { background-color : red; }
  • Method 3: Write your style inline with your HTML

<body> <div style="background-color: red; color: white"> This div will have a red background and white text. </div>

11. The Box

We can interact with page space in three diffeent levels. Each of these can be customized in different ways.

  • Margin. Refers to the space around all objects in our page. It is transparent without color. It is the space between boxes.
  • Border. As its name this refers to a line that we can customize around the content. There are new models arriving that allows to control more precissely the size of border and padding. This is a suggested code to make visualize and control under this new model:
* {
outline: 1px solid red !important;
}
* {
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
}

This make references to different browsers rules to make this work with them.

  • Padding: Padding is a space we can give our content to separate it from the border.
  • Content: Refers to the space used by text, images, video or any element we add to our page.

12. Flexbox

Divs are block elements (as opposed to inline), so by default they take up the entire width of a page. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other. We can create different and interactive possition for our objects using Flex boxes. This allows objects to show together when space allows it ignoring the block rule. A Guide for flexbox properties can be found in the next link.


Flex Guide

Here is one way to activate this.

.app { display: -webkit-flex; display: flex; }
